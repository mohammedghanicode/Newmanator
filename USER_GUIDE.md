# 📘 Newmanator — User Guide

Welcome to **Newmanator**, the desktop and web utility for collating API test reports generated by **Postman + Newman**. This guide explains how to install, run, and use Newmanator effectively.

---

## 🚀 What Newmanator Does

Newmanator helps you:

- Upload or select multiple Newman report files
- Extract and combine results from `.zip` archives and `.html` reports
- Generate a single interactive `summary.html`
- Open the summary automatically in your browser

It supports both the Electron desktop interface and a local web server.

---

## ✅ Prerequisites

- Windows, macOS, or Linux
- Node.js installed (recommended version 18+)
- PowerShell available for the desktop launcher and script flow

---

## 📦 Installation

1. Clone the repository:

```powershell
git clone https://github.com/mohammedghanicode/Newmanator.git
cd Newmanator
```

2. Install dependencies:

```powershell
npm install
```

---

## 🧩 Supported Report Formats

Newmanator accepts:

- `.zip` archives containing Newman report files
- `.html` Newman HTML report files
- `.htm` report files

For `.zip` archives, Newmanator extracts the contents into `unzipped/` before summarizing.

---

## ▶️ Usage Modes

### 1. Desktop App (Electron)

Use this mode when you want a simple graphical workflow.

```powershell
npm start
```

Then:

- Select one or more `.zip` / `.html` files
- Click **Process_Reports**
- Wait for processing to finish
- Open the generated `summary.html`

### 2. Web UI

Run the built-in web server and upload files through a browser.

```powershell
npm run start:web
```

Open the browser at:

- `http://localhost:3000`

Then:

- Upload `.zip` or `.html` files
- Track progress
- Download the summary report when complete

### 3. PowerShell Script

For command-line driven use, run the built-in script.

```powershell
npm run powershell
```

This launches a file picker. Select your report files and the script will:

- Extract `.zip` archives if needed
- Generate `summary.html`
- Open the report automatically when done

---

## 🧪 What Gets Processed

### PowerShell / Desktop file handling

- `.zip` files are processed with `process-zip.js`
- Mixed `.html` and multiple `.zip` inputs are handled by `process-files.js`
- `summarise.js` generates the final summary from `unzipped/`

### Output location

- In development, output appears in the repository folder
- In packaged Electron mode, output is written under the app `userData` folder
- The final file is always named `summary.html`

---

## 🗂️ Recommended Zip Structure

To avoid errors, keep your archive organized like this:

```text
my-newman-reports.zip
├── run-001/
│   ├── report.html
│   └── report.json
├── run-002/
│   ├── report.html
│   └── report.json
└── run-003/
    └── report.html
```

If only HTML files are present, Newmanator will still parse them.

## 🖥️ Desktop App User Guide

### Main Workflow

1. Click the report selection drop zone
2. Add `.zip` or `.html` files
3. Review the selected files list
4. Click **Process_Reports**
5. Wait for progress and logs
6. Click **Open Summary** if available

### Key Controls

- `Process_Reports`: Starts report collation
- `Clear_Files`: Clears the current file queue
- `Open_Summary_Report`: Opens the generated `summary.html`
- `SELECTED_REPORTS`: Shows file names and sizes
- `Terminal_Logs`: Displays execution output from PowerShell

## 📊 What the Summary Contains

The generated `summary.html` typically includes:

- Overall test counts
- Pass / fail totals
- Test completion status
- Request details for failures
- Grouped report summaries

The report is created by extracting Newman report data and converting it into a single aggregated HTML page.

## ⚠️ Troubleshooting

### Common issues

- `No valid .zip or .html files found` — Ensure you selected supported file types
- `summary.html was not generated` — Check for extraction errors and that `unzipped/` has valid report contents
- `Unsupported file type` — Only `.zip`, `.html`, and `.htm` are accepted
- `Too many concurrent processing sessions` (web mode) — Wait and retry, or increase `MAX_CONCURRENT_SESSIONS`

### Tips

- Keep file names simple and avoid nested archives where possible
- Use `.zip` archives if you need to process multiple reports at once
- If summary generation fails, inspect the terminal logs or browser logs for exact error details

---

## 🔍 Developer Notes

### How processing works

1. `run-reporter.ps1` collects selected inputs
2. `process-zip.js` extracts a single archive
3. `process-files.js` handles mixed inputs and multi-file workflows
4. `summarise.js` generates the final `summary.html`

### Important folders

- `unzipped/` — extracted report contents
- `uploads/` — uploaded files in web mode
- `results/` — generated results in web mode

When running from terminal, you can use:

## 🐛 Troubleshooting

### "Error invoking remote method 'process-files'"

**Cause:** File paths or working directories are invalid in production build.

**Solution:**

1. Ensure all required folders exist (`uploads/`, `results/`, `unzipped/`)
2. Check file permissions on your user data folder
3. View logs in the terminal to see the exact error

### Reports Not Found After Processing

**Cause:** Summary file not generated or stored in wrong location.

**Solution:**

- Check the terminal logs for extraction errors
- Verify zip file contains valid Newman reports
- Ensure reports have both `.html` and `.json` files when possible

### Performance Issues with Large Zips

**Cause:** Processing many reports at once consumes memory.

**Solution:**

- Split large zip files into smaller batches
- Close other applications
- Use the web version which handles memory better: `npm run start:web`

### Files Disappear After Restart

**Cause:** Data folders were excluded from packaged build.

**Solution:** This is expected behavior in production to keep the app size small. Data is stored in:

- Development: `./unzipped/`, `./results/`
- Production: `%AppData%\Newmanator\` (Windows user data folder)

## 📈 Best Practices

### 1. Organize Reports Before Processing

- Use consistent naming conventions: `collection-env-timestamp.zip`
- Keep reports in dated folders: `2025-05-13/test-reports.zip`

### 2. Archive Your Summaries

- Save generated `summary.html` reports to version control or network drive
- Reference them later for trend analysis

### 3. Use Batch Processing

- Combine similar test runs in a single zip
- Avoid processing single reports repeatedly

### 4. Monitor Performance Metrics

- Use the summary report to identify slow API endpoints
- Share findings with backend teams

### 5. Automate with CI/CD

- Download reports from your CI runner (Jenkins, GitHub Actions, Azure Pipelines)
- Pipe them into Newmanator as part of post-test steps

## 🔐 Security & Privacy

### Sensitive Data Redaction

Newmanator automatically redacts:

- Bearer tokens and JWT tokens
- API keys and credentials
- Authorization headers
- Long base64 strings (possible secrets)

### Safe to Share

- Generated `summary.html` is safe to share with stakeholders
- No credentials are exposed in reports
- Only aggregate stats and sanitized failure data included
